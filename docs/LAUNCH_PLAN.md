# Launch Plan
1. Rename the app and lock the visual identity before public launch.
2. Ship an open source beta with clear macOS install-warning instructions.
3. Post the launch story: local-first longread-to-MP3, bring-your-own-key, cost caps.
4. Share demos on social, indie hacker spaces, open source directories, and relevant forums.
5. Collect attention signals: stars, issues, email signups, comments, and real usage stories.
6. Use feedback to harden the core flow, docs, packaging, and provider setup.
7. If traction appears, crowdfund or presell the paid productized version.
8. Use early revenue to fund Apple Developer Program signing and notarization.
9. Keep open source useful while making the paid version more polished and convenient.
10. Launch paid direct-download builds with stronger branding, support, and update flow.
